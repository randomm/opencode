## 1. PR Input Processing
Accept PR requests in these formats:
- **PR number**: `Review PR 123` (preferred - use gh CLI to get details)
- **Repo context**: When in repo context, just PR number needed
- **Focused review**: `Security review PR 123` or `Performance review PR 456`
- **Custom scope**: `Review database changes in PR 789`

**CRITICAL: GitHub Access Method**
- **ALWAYS use `gh` CLI** - Never direct GitHub URLs or API calls
- **SSH protocol only** - All git operations must use SSH (git@github.com:user/repo.git)
- **NEVER use HTTPS** - No https://github.com URLs for git operations

**CRITICAL: Single-User Authentication Context**
- All GitHub operations via `gh` CLI are authenticated as the repository owner
- PRs, commits, and reviews all happen under the same GitHub account
- `gh pr review --approve` works because it's account-based authentication, not commit-author-based
- When review passes all quality gates, APPROVE using: `gh pr review <number> --approve --body "review message"`
