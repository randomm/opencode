## **CI/CD MONITORING - MANDATORY**

### **CRITICAL: NEVER ASSUME CI WILL PASS**
- When you push commits, you MUST monitor the CI run
- Use `gh run list` and `gh run view` to check status
- Poll every 30-60 seconds until completion
- Report actual results, not predictions

### **CI Monitoring Protocol:**
1. After pushing, immediately get the workflow run ID
2. **Real-time monitoring**: Use `gh run watch [run-id]` for live status updates
3. **Alternative polling**: `watch -n 30 'gh run view [run-id] --json status,conclusion'`
4. **Log streaming**: `gh run view [run-id] --log-failed` for failure details
5. Monitor until status is "completed" and conclusion is determined
6. Report back with ACTUAL results, not assumptions

### **Enhanced CI Commands:**
- `gh run watch [run-id]` - Real-time run monitoring with live updates
- `watch -n 30 'gh run list --limit 1'` - Monitor latest run every 30 seconds  
- `gh run view [run-id] --log` - Stream complete logs
- `gh run view [run-id] --log-failed` - Only failed job logs
- `gh run rerun [run-id]` - Restart failed runs when appropriate

### **FORBIDDEN Responses:**
- ❌ "CI should pass"
- ❌ "CI will likely pass"
- ❌ "Everything looks good for CI"
- ❌ "CI is expected to pass"

### **REQUIRED Responses:**
- ✅ "CI run #123 started - monitoring..."
- ✅ "CI run #123 in progress (2 min elapsed)..."
- ✅ "CI run #123 PASSED ✅ (took 3m 45s)"
- ✅ "CI run #123 FAILED ❌ - [specific error]"

### **Example Workflow:**
```bash
# Push changes
git push

# Get latest run
run_id=$(gh run list --limit 1 --json databaseId -q '.[0].databaseId')

# Method 1: Real-time monitoring (preferred)
gh run watch $run_id

# Method 2: Custom watch command for detailed monitoring
watch -n 30 "gh run view $run_id --json status,conclusion,startedAt | jq -r '.status + \" (\" + .conclusion + \")\"'"

# Method 3: Background monitoring with notifications
watch -n 30 'gh run list --limit 1 --json status,conclusion | jq -r ".[0].status + \" (\" + (.[0].conclusion // \"running\") + \")\""' &

# After completion, get detailed results
if [ "$(gh run view $run_id --json conclusion -q '.conclusion')" = "success" ]; then
  echo "CI run #$run_id PASSED ✅"
else
  echo "CI run #$run_id FAILED ❌"
  gh run view $run_id --log-failed
fi
```

### **Task Completion Rule:**
NO task is complete until CI verification is done. The user should never have to ask "did CI pass?" - you should already know and report it.
