**Autonomous Workflow Process:**

1. **Initial Assessment**
   - Run `git status` to understand current state
   - Check branch with `git branch --show-current`
   - Verify remote status with `git remote -v` and `git fetch --dry-run`
   - Use `gh repo view` to understand repository context

2. **Intelligent Change Analysis**
   - Use `git diff --stat` to assess scope of changes
   - Group related changes for atomic commits
   - Identify files that should be in separate commits
   - Check for sensitive data or credentials

3. **Commit Strategy**
   - Create semantic commits following conventional commits standard:
     * feat: new features
     * fix: bug fixes
     * docs: documentation changes
     * refactor: code restructuring
     * test: test additions/modifications
     * chore: maintenance tasks
     * perf: performance improvements
     * style: formatting changes
   - Keep commits atomic and focused
   - Write clear, descriptive messages (50 char subject, detailed body if needed)

4. **Branch Management**
   - Follow naming conventions:
     * feature/description
     * bugfix/issue-number-description
     * hotfix/critical-fix
     * release/version
   - Keep branches up-to-date with base branch
   - Handle rebasing when appropriate

5. **GitHub Integration**
   - Create PRs with comprehensive descriptions using `gh pr create`
   - Link issues automatically with "Fixes #123" syntax
   - Set up draft PRs for work in progress
   - Request reviews from appropriate team members
   - Monitor CI/CD status with `gh run list` and `gh pr checks`

**Advanced Git Operations:**

```bash

```
