**Core React Development Principles:**

1. **Component Architecture**
   - Design with composition over inheritance
   - Create small, focused, reusable components
   - Follow single responsibility principle
   - Use proper component naming (PascalCase)
   - Organize components by feature/domain
   - Implement proper prop drilling prevention
   - Use compound component patterns when appropriate

2. **TypeScript Excellence**
   - Use strict TypeScript configuration
   - Define comprehensive interfaces for all props
   - Avoid using 'any' type - use unknown or generics
   - Create reusable type utilities
   - Use discriminated unions for complex state
   - Implement proper type guards
   - Export types alongside components

3. **State Management Best Practices**
   - Use appropriate state solutions:
     * useState for local component state
     * useContext for cross-component state
     * useReducer for complex state logic
     * External libraries (Redux/Zustand) for app-wide state
   - Keep state as local as possible
   - Normalize complex state structures
   - Implement optimistic updates properly

4. **React Hooks Mastery**
   - Follow Rules of Hooks strictly
   - Create custom hooks for reusable logic
   - Use proper hook dependencies
   - Implement cleanup in useEffect
   - Optimize with useMemo and useCallback appropriately
   - Avoid unnecessary re-renders
   - Document custom hooks thoroughly

5. **Testing Standards**
   **MANDATORY before delegating to @git-autonomous-agent:**
   - Achieve minimum 80% test coverage
   - Test types to implement:
     * Unit tests with Jest
     * Component tests with React Testing Library
     * Integration tests for feature flows
     * Accessibility tests with jest-axe
     * Snapshot tests for stable components
   - Follow testing best practices:
     * Test behavior, not implementation
     * Use proper queries (getByRole > getByText)
     * Mock external dependencies appropriately
     * Test error states and edge cases

6. **Code Quality and Linting**
   **FIRST: Discover project linters** - Check package.json scripts, .eslintrc, .prettierrc, tsconfig.json
   **CRITICAL: Run before delegating to @git-autonomous-agent:**
   - ESLint with React and TypeScript rules
   - Prettier for consistent formatting
   - TypeScript compiler checks (tsc --noEmit)
   - Jest test suite
   - Bundle size analysis
   - Accessibility audit tools

7. **Performance Optimization**
   - Implement React.memo for expensive components
   - Use useMemo for expensive computations
   - Use useCallback for stable function references
   - Implement code splitting with lazy/Suspense
   - Optimize bundle size:
     * Tree shaking
     * Dynamic imports
     * Analyze with webpack-bundle-analyzer
   - Use React DevTools Profiler
   - Implement virtual scrolling for long lists

8. **Accessibility (a11y) Standards**
   - Ensure WCAG 2.1 AA compliance minimum
   - Use semantic HTML elements
   - Implement proper ARIA attributes
   - Ensure keyboard navigation works
   - Test with screen readers
   - Provide proper focus management
   - Include skip links and landmarks
   - Test color contrast ratios

9. **Styling Best Practices**
   - Use consistent styling approach:
     * CSS Modules for scoped styles
     * Styled Components for dynamic styles
     * Tailwind for utility-first approach
   - Implement responsive design
   - Use CSS custom properties for theming
   - Optimize critical CSS delivery
   - Avoid inline styles except for dynamic values

10. **Form Handling**
    - Use controlled components appropriately
    - Implement proper validation:
      * Client-side with libraries (React Hook Form, Formik)
      * Display clear error messages
      * Implement proper error boundaries
    - Handle form submission states
    - Implement proper accessibility for forms
    - Use proper input types and attributes
