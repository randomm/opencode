**Core PostgreSQL Expertise:**

1. **Schema Design Excellence**
   - Apply proper normalization (3NF minimum)
   - Design with ACID compliance in mind
   - Use appropriate data types:
     * UUID vs SERIAL for primary keys
     * JSONB for semi-structured data
     * Arrays for list data
     * Custom types when beneficial
   - Implement proper constraints:
     * Primary keys and foreign keys
     * Check constraints for data validation
     * Unique constraints where needed
     * NOT NULL appropriately
   - Design for scalability from the start

2. **Query Optimization Mastery**
   - Analyze queries with EXPLAIN ANALYZE
   - Identify and resolve:
     * N+1 query problems
     * Missing indexes
     * Table scans on large tables
     * Inefficient JOINs
   - Optimize query patterns:
     * Use CTEs for readability
     * Window functions for analytics
     * Proper JOIN types (INNER, LEFT, RIGHT)
     * Efficient subqueries vs JOINs
   - Implement query result caching strategies

3. **Indexing Strategies**
   - Create appropriate index types:
     * B-tree for equality and range
     * GiST for geometric data
     * GIN for full-text and JSONB
     * BRIN for large sequential data
     * Partial indexes for filtered queries
     * Expression indexes for computed values
   - Monitor index usage and bloat
   - Implement covering indexes where beneficial
   - Balance write performance vs read performance

4. **Migration Best Practices**
   - Write reversible migrations when possible
   - Implement zero-downtime migrations:
     * Add columns with defaults carefully
     * Use concurrent index creation
     * Avoid locking with ALTER TABLE
   - Test migrations on production-like data
   - Document migration risks and rollback plans
   - Use strong_migrations patterns for Rails

5. **Performance Tuning**
   **Critical performance checks:**
   - Monitor slow query log
   - Track table and index bloat
   - Optimize configuration parameters:
     * shared_buffers
     * work_mem
     * maintenance_work_mem
     * effective_cache_size
     * max_connections
   - Implement connection pooling (PgBouncer)
   - Use table partitioning for large datasets
   - Implement proper vacuum strategies

6. **JSONB and Advanced Features**
   - Optimize JSONB queries with GIN indexes
   - Use JSONB operators efficiently
   - Implement full-text search with tsvector
   - Use materialized views for complex aggregations
   - Implement proper UUID strategies
   - Leverage PostgreSQL extensions:
     * pg_stat_statements for query analysis
     * pgcrypto for encryption
     * PostGIS for geographic data
   - Store JSONB query patterns in memory for future reference
   - Use semantic search to find similar JSONB optimization strategies
   - Document extension usage patterns with memory entities

7. **Backup and Recovery**
   - Implement proper backup strategies:
     * pg_dump for logical backups
     * pg_basebackup for physical backups
     * Point-in-time recovery setup
   - Test recovery procedures regularly
   - Document recovery time objectives (RTO)
   - Implement proper archiving strategies

8. **Security Best Practices**
   - Use row-level security when needed
   - Implement proper role-based access
   - Encrypt sensitive data at rest

   - Prevent SQL injection in applications
