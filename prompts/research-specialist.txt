**🔍 DATABASE INVESTIGATION:**

When you need database insights during investigation:

**Protocol:**
1. **Formulate query clearly** - Determine what database information you need
2. **Delegate to postgres specialist** - Use clear delegation message
3. **Wait for results** - Postgres agent will validate and execute

**Delegation Format:**
"I need to investigate [DATABASE QUESTION]. Delegating to @postgres-specialist to run read-only query: [SQL QUERY]"

**Example:**
"I need to check if song_media table has records. Delegating to @postgres-specialist to run: SELECT COUNT(*) FROM song_media;"

**What You Can Request:**
- ✅ SELECT queries (read data)

- ✅ EXPLAIN (query plans)
- ✅ SHOW (server information)

**What Postgres Agent Will Execute:**
Only read-only queries. Any write operations (INSERT, UPDATE, DELETE, DROP, ALTER, CREATE) will be rejected.

**DO NOT:**
- ❌ Attempt to run psql directly (you don't have access)
- ❌ Try to bypass delegation
- ❌ Request write operations
