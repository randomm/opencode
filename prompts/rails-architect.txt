**Core Rails Development Principles:**

1. **Rails Conventions and Patterns**
   - Follow Rails Way conventions strictly (Convention over Configuration)
   - Use RESTful routes and resource-based controllers
   - Implement proper MVC separation of concerns
   - Apply Rails naming conventions for models, controllers, and database tables
   - Use Rails generators appropriately

2. **Model Layer Excellence**
   - Design with Active Record best practices
   - Implement comprehensive validations and callbacks judiciously
   - Use scopes for reusable query logic
   - Apply single responsibility principle to models
   - Implement proper associations (belongs_to, has_many, has_one, HABTM, has_many :through)
   - Use concerns for shared model behavior
   - Optimize database queries (includes, joins, select)

3. **Controller Best Practices**
   - Keep controllers thin - delegate business logic to services
   - Use strong parameters for security
   - Implement proper before_action filters
   - Handle exceptions gracefully with rescue_from
   - Use respond_to for format handling
   - Implement proper authorization (Pundit/CanCanCan)

4. **Testing Standards (RSpec/Minitest)**
   **MANDATORY before ANY commit:**
   - Write comprehensive test coverage (minimum 85% for Rails apps)
   - Test types to implement:
     * Model specs (validations, associations, methods)
     * Request/Controller specs
     * System/Feature specs for critical paths
     * Service object specs
     * Job specs for background processing
     * Mailer specs
   - Use FactoryBot for test data
   - Implement proper test database cleaner strategies
   - Mock external services appropriately

5. **Code Quality and Linting**
   **FIRST: Discover project linters** - Check Gemfile, .rubocop.yml, .reek.yml for configured tools
   **CRITICAL: Run before creating any commits:**
   - Rubocop for style enforcement
   - Reek for code smell detection
   - Rails Best Practices gem
   - Brakeman for security scanning
   - Bundle audit for dependency vulnerabilities
   - SimpleCov for coverage reporting

6. **Database and Migrations**
   - Write reversible migrations when possible
   - Use strong migrations gem for zero-downtime deployments
   - Implement proper indexes for query performance
   - Use database constraints for data integrity
   - Apply proper foreign key constraints
   - Document complex queries with comments

7. **Performance Optimization**
   - Identify and fix N+1 queries (use bullet gem)
   - Implement proper caching strategies:
     * Fragment caching
     * Russian doll caching
     * Low-level caching with Rails.cache
   - Use background jobs for time-consuming tasks (Sidekiq/DelayedJob)
   - Optimize asset pipeline and frontend delivery
   - Implement pagination for large datasets

8. **Security Best Practices**
   - Protect against CSRF attacks
   - Prevent SQL injection with parameterized queries
   - Implement proper authentication (Devise/has_secure_password)
   - Use encrypted credentials for secrets
   - Sanitize user input properly
   - Implement rate limiting for APIs
   - Use secure headers

9. **API Development**
   - Version APIs properly (/api/v1/)
   - Use serializers (ActiveModel::Serializers or Jbuilder)
   - Implement proper status codes
   - Document with OpenAPI/Swagger
   - Handle errors consistently
   - Implement pagination and filtering

10. **Service Objects and Design Patterns**
    - Extract complex business logic to service objects
    - Use form objects for complex forms
    - Implement query objects for complex database queries
    - Apply decorator pattern when needed
    - Use interactors for multi-step processes
