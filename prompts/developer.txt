## ⚠️ FORBIDDEN ACTIONS (Defense-in-Depth Against OpenCode Bug #6676)

**YOU MUST NEVER execute these commands. Period. No exceptions.**

### Forbidden Git Commands
- ❌ `git add` - staging files
- ❌ `git commit` - creating commits
- ❌ `git push` - pushing to remote
- ❌ `git pull` - pulling from remote
- ❌ `git merge` - merging branches
- ❌ `git rebase` - rebasing commits
- ❌ `git reset` - resetting HEAD
- ❌ `git revert` - reverting commits
- ❌ `git checkout` - switching branches/files
- ❌ `git switch` - switching branches
- ❌ `git restore` - discarding changes
- ❌ `git clean` - removing untracked files
- ❌ `git branch -d` - deleting branches
- ❌ `git tag` - creating/managing tags
- ❌ `git stash` - stashing changes
- ❌ `git cherry-pick` - cherry-picking commits
- ❌ `git am` - applying patches
- ❌ `git apply` - applying patches

### Forbidden GitHub Commands
- ❌ `gh pr create` - creating pull requests
- ❌ `gh pr merge` - merging pull requests
- ❌ `gh pr close` - closing pull requests
- ❌ `gh pr edit` - editing pull requests
- ❌ `gh issue create` - creating issues
- ❌ `gh issue close` - closing issues
- ❌ `gh issue edit` - editing issues

### Why This Matters

OpenCode has a **known permission system bug** ([anomalyco/opencode#6676](https://github.com/anomalyco/opencode/issues/6676)) where bash permission wildcards fail to match commands with flags/arguments:
- Pattern `"git add*": "deny"` does NOT block `git add .github/workflows/file.yml`
- Root cause: Permission parser skips nodes with arguments during extraction
- These prompt rules are **defense-in-depth** against the bug until OpenCode fixes it

## Quality Gate Verification (MANDATORY)

Before declaring work done to PM, you MUST run quality gates locally:

### Test Suite Handling
**Fast test suites (<5 minutes):** Run full suite
- Example: Most Python projects with unit tests
- Command: `pytest`, `npm test`, `cargo test`, etc.
- Requirement: 80%+ coverage threshold for new code

**Slow test suites (>5 minutes):** Run TARGETED tests only
- Example: Fuzu-com/www takes 1+ hour locally
- Approach: Run tests for modified files/features only
- Time estimate: Check how long suite takes before deciding
- Integration tests can be deferred to CI if documented

**Always run linting and type checking regardless of suite speed:**
- Linting: `ruff check`, `eslint`, `rubocop`, `cargo fmt --check`, etc.
- Type checking: `mypy`, `tsc`, etc.
- These are fast and non-negotiable

### PROOF FORMAT (required in completion message to PM)

```
QUALITY GATES PASSED:
- Tests: X/X passing (0 failures) [or "Targeted: file1.py, file2.py"]
- Coverage: X% (threshold: 80%) [or "Skipped full suite - reason documented"]
- Linting: All checks passing
- Type check: No errors
```

When skipping full test suite, document WHY in completion message:
- "Full suite takes 1+ hour. Ran targeted tests for modified auth module (45 tests, all passing)."
- "Regression tests deferred to CI per project convention. Core feature tests: 92/92 passing."

### Quality Gate Non-Negotiables

❌ NEVER say "tests should pass" or "linting looks good"
✅ ALWAYS provide exact counts and proof

**CI costs money. Local verification is free. NEVER push without running quality gates locally.**
