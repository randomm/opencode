**Core API Design Principles:**

1. **RESTful Design Excellence**
   - Follow REST architectural constraints:
     * Client-server separation
     * Statelessness
     * Cacheability
     * Uniform interface
     * Layered system
   - Use proper HTTP methods:
     * GET for reading (idempotent)
     * POST for creation
     * PUT for full updates (idempotent)
     * PATCH for partial updates
     * DELETE for removal (idempotent)
   - Design resource-oriented URLs:
     * /api/v1/users (collection)
     * /api/v1/users/{id} (resource)
     * /api/v1/users/{id}/posts (nested resource)

2. **API Versioning Strategies**
   - Implement versioning from day one
   - Choose appropriate versioning method:
     * URL path versioning (/api/v1/)
     * Header versioning (Accept: application/vnd.api+json;version=1)
     * Query parameter (?version=1)
   - Maintain backward compatibility
   - Document deprecation policies
   - Provide migration guides

3. **Authentication & Authorization**
   - Implement secure authentication:
     * JWT for stateless auth
     * OAuth 2.0 for third-party access
     * API keys for service-to-service
   - Design proper authorization:
     * Role-based access control (RBAC)
     * Attribute-based access control (ABAC)
     * Resource-level permissions
   - Implement refresh token rotation
   - Use secure token storage practices

4. **Request/Response Design**
   - Design consistent request formats:
     * Use JSON as default
     * Support content negotiation
     * Validate input thoroughly
   - Standardize response structures:
     ```json
     {
       "data": {},
       "meta": {
         "timestamp": "2024-01-01T00:00:00Z",
         "version": "1.0"
       },
       "links": {
         "self": "/api/v1/resource/1",
         "next": "/api/v1/resource/2"
       }
     }
     ```
   - Implement proper pagination:
     * Cursor-based for real-time data
     * Page-based for static data
     * Include total count and links

5. **Error Handling Standards**
   - Use appropriate HTTP status codes:
     * 200 OK - Successful GET/PUT
     * 201 Created - Successful POST
     * 204 No Content - Successful DELETE
     * 400 Bad Request - Client error
     * 401 Unauthorized - Authentication required
     * 403 Forbidden - Authorization failed
     * 404 Not Found - Resource doesn't exist
     * 422 Unprocessable Entity - Validation errors
     * 429 Too Many Requests - Rate limited
     * 500 Internal Server Error - Server error
   - Provide detailed error responses:
     ```json
     {
       "error": {
         "code": "VALIDATION_ERROR",
         "message": "Validation failed",
         "details": [
           {
             "field": "email",
             "message": "Invalid email format"
           }
         ]
       }
     }
     ```

6. **GraphQL Design (when applicable)**
   - Design efficient schemas:
     * Avoid N+1 queries with DataLoader
     * Implement proper pagination (Relay spec)
     * Use fragments for reusability
   - Implement proper error handling
   - Design with performance in mind:
     * Query complexity analysis
     * Depth limiting
     * Rate limiting by complexity

7. **API Documentation**
   - Use OpenAPI/Swagger specification
   - Document every endpoint:
     * Description and purpose
     * Request parameters and body
     * Response formats and examples
     * Error responses
     * Rate limits
   - Provide interactive documentation
   - Include authentication examples
   - Maintain changelog

8. **Performance Optimization**
   - Implement caching strategies:
     * HTTP caching headers (ETag, Cache-Control)
     * CDN integration
     * Application-level caching
   - Optimize response times:
     * Database query optimization
     * Response compression (gzip)
     * Field filtering (?fields=id,name)
     * Eager loading relationships
   - Implement rate limiting:
     * Per-user limits
     * Endpoint-specific limits
     * Graceful degradation

9. **Security Best Practices**
   - Implement HTTPS everywhere
   - Use proper CORS configuration
   - Validate and sanitize all inputs
   - Implement rate limiting and throttling
   - Use API gateways for additional security
   - Log and monitor suspicious activity
   - Implement request signing for sensitive operations
   - Use field-level encryption for PII

10. **Testing Strategies**
    - Write comprehensive API tests:
      * Unit tests for business logic
      * Integration tests for endpoints
      * Contract tests for API consumers
      * Load tests for performance
    - Test error scenarios thoroughly
    - Validate response formats
    - Test authentication and authorization
    - Implement API monitoring
